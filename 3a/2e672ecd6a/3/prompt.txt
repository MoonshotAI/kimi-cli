# Fix: CLI 中 FetchURL 截断 URL 支持 Ctrl+Click 打开完整链接

## Context

Web 端直接从 `toolCall.input.url` 读取完整 URL 用于超链接，与 `primaryParam`（截断文本）分离。CLI 端应采用相同模式：从 `self._lexer`（原始 JSON）提取完整 URL，不修改 `extract_key_argument`。

## 方案

#### `_ToolCallBlock` 新增私有静态方法

```python
@staticmethod
def _extract_full_url(json_content: str | streamingjson.Lexer, tool_name: str) -> str | None:
    if tool_name != "FetchURL":
        return None
    json_str = json_content.complete_json() if isinstance(json_content, streamingjson.Lexer) else json_content
    try:
        args = json.loads(json_str)
    except (json.JSONDecodeError, ValueError):
        return None
    if isinstance(args, dict) and args.get("url"):
        return str(args["url"])
    return None
```

等价于 web 端的：
```typescript
const fullUrl = rawName === "FetchURL" && input?.url ? input.url : null;
```

#### `__init__` 新增 `self._full_url`


#### `append_args_part` 同步更新

#### `_get_headline_markup()` → `_build_headline_text()` 返回 `Text`

截断文本通过 `Style(link=full_url)` 附加 OSC 8 超链接（仅 FetchURL）：

```python
def _build_headline_text(self, ...) -> Text:
    text = Text()
    text.append("Used " if finished else "Using ")
    text.append(tool_name, style="blue")
    if argument:
        text.append(" (", style="grey50")
        arg_style = Style(color="grey50", link=full_url) if full_url else "grey50"
        text.append(argument, style=arg_style)
        text.append(")", style="grey50")
    return text
```
