# 附录 C：术语表

## A

**Agent（代理）**
: 能够感知环境、自主决策并采取行动的 AI 系统。

**ACP (Agent Client Protocol)**
: Agent 客户端协议，用于 IDE 集成。

**Approval（审批）**
: 在执行某些操作前请求用户确认的机制。

## C

**Context（上下文）**
: Agent 的"记忆"，包含对话历史和状态。

**Checkpoint（检查点）**
: 上下文的保存点，用于时间旅行功能。

## D

**D-Mail**
: 时间旅行消息，可以发送到之前的检查点。

**DenwaRenji（电话微波炉）**
: Kimi-CLI 中的时间旅行系统，名字来自《命运石之门》。

## F

**Function Calling**
: LLM 的功能，允许 AI 调用预定义的函数/工具。

## K

**KAOS**
: Kimi Agent Operating System，文件系统抽象层。

**KimiSoul**
: Agent 的执行引擎，负责主循环。

## L

**LLM (Large Language Model)**
: 大语言模型，如 GPT-4、Claude 等。

**LaborMarket（劳动市场）**
: 子 Agent 的注册表。

## S

**Soul（灵魂）**
: Agent 的执行引擎协议。

**System Prompt（系统提示词）**
: 定义 Agent 行为和能力的指令。

**Subagent（子 Agent）**
: 专门处理特定任务的 Agent。

## T

**Tool（工具）**
: Agent 能够执行的操作，如读文件、执行命令等。

**Thinking Mode（思维模式）**
: 允许 Agent 进行更深入推理的模式。
